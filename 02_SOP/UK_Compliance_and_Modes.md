# UK Master 合规层与三模式运行（降风险改造）

> 本文档将 `Master_Coursework_OS` 定位为：**学习与写作管理系统**，而非 AI 代写流水线。  
> 政策引用类型：曼大 AI guidance、学术不端/plagiarism 指南（见用户复盘中的 documents.manchester.ac.uk / staffnet / TEL 链接）。**每门课以 module 最新 handbook 为准。**

**Coursework planning, evidence management, AI-assisted review, and integrity control system.**

不叫「AI 帮我完成作业 SOP」。

## 3. 三模式定义

### Mode A — No-AI / Minimal-AI

**用于**：禁止 AI 的 assessment；take-home exam（除非明确允许）；高风险个人反思；unaided work。

| 允许 | 禁止 |
| --- | --- |
| 自己读题、写作 | 生成式 AI 写段落 |
| 拼写/语法工具（若 policy 允许） | ChatGPT 出 outline/论点 |
| 学校指定工具（若有） | 外部 AI 上传课程材料 |

**OS 裁剪**：只用 rubric 表、文献表、checklist、文件结构；**跳过** ChatGPT 写作类 prompt。

---

### Mode B — AI-supported learning（默认）

**用于**：essay、report、literature review、presentation、proposal、多数 technical report。

| AI 可做 | 你必须做 |
| --- | --- |
| 解释概念、outline 草案、检索关键词 | 中心论点与取舍 |
| 批判草稿、逻辑漏洞 | 文献检索与 Verified |
| 语言清晰化（在你文稿上） | declaration / 诚信 |
| 终检对照 brief（不背书文献） | Learning checkpoint |

**OS 使用**：完整 SOP，但 `04_outline` 产出须经你删改；正文核心段亲手写。

---

### Mode C — AI-integrated task

**用于**：明确要求 AI 协作/反思；coding/data 允许 AI；类似本次南邮 50% AI 记录。

| 特点 |
| --- |
| 完整 `ai_log`、比例声明、六段式反思 |
| 仍可遵守：不编造文献、不伪造交互 |

**Galison_Assignment** 即 Mode C；**不要默认**英硕每课都是 Mode C。

## 4. Task 类型适配（摘要）

| 类型 | 建议模式 | 备注 |
| --- | --- | --- |
| Essay / Report | B | |
| Reflective | B 或 A | 内容必须真实，AI 只帮结构 |
| Literature review | B | 文献表核心 |
| Technical paper | B | LaTeX 流程保留 |
| Take-home exam | A | 先查 policy |
| Group work | B + 隐私规则 | 不上传组员未授权内容 |
| Dissertation | B | 数据/伦理问 supervisor |
| Lab report | B/A | AI 不捏造数据 |

| 风险 | 应对（文件/步骤） |

## 7. 与 Galison 项目的区别（避免照搬）

| Galison（Mode C） | 英硕默认（Mode B） |
| --- | --- |
| AI 记录 50 分、6000 字 | 可能只需短 declaration |
| 鼓励上传指南 PDF 给 AI | 优先摘要、学校工具 |
| ChatGPT 英文段落草稿多 | _critique_ 为主，草稿亲手写 |
| 25h 双 PDF 工程 | 按 credit 缩放时间 |

Galison 复盘仍见 `Galison_Assignment/99_Project_Review_Assets/`；**执行用本 OS + 本合规层**。
